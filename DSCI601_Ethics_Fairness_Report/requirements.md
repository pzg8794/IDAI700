# DSCI601 Ethics and Fairness Report Requirements

## Exact assignment requirements

Source:
- `Ethics and Fairness Report (Individual Assignment) - DSCI.601.01 - Applied Data Science I - RIT myCourses.pdf`

The assignment requires:

- individual work, not group work
- a one-page document in Overleaf
- selection of one or more of:
  - ethics
  - privacy
  - fairness
  - security
- an explanation of how the selected area relates to the project
- an explanation of how addressing the selected area has changed or would change:
  - project design
  - implementation
  - ability to share data so others can reproduce the work

Provided supporting readings:
- Inherent Limitations of AI Fairness
- AAAI/AIES paper linked in the assignment
- Alan Turing Institute fairness guide linked in the assignment

## Project-specific interpretation for this report

This report should focus primarily on `fairness`, with `ethics` used as the framing layer.

Why:

- the current DSCI601 project already centers fairness-aware sequential decision-making
- the project spans two domains where group-level disparities matter:
  - clinical diagnostic workflows
  - quantum routing and resource allocation
- the older IDAI700 ethics paper gives a clear motivation for why fairness should be treated as
  enforceable and structural rather than cosmetic

## Required content for the final one-page draft

The final one-page report must explicitly cover:

1. what the current DSCI601 project is
2. why fairness is a central concern in this project
3. how fairness changes or would change design and implementation
4. how fairness changes or would change data sharing and reproducibility

## Constraints for the draft

- keep the tone research-facing and practical
- do not turn the report into a full literature review
- do not reuse the older IDAI700 paper verbatim
- use the older paper only as motivation and conceptual grounding
